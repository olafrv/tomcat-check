Overview

tomcat-check is Linux perl script to monitor periodically Apache Tomcat processes status through some custom checks, comparing border conditions against data generated by the Apache Tomcat built-in Manager, the psi-probe improved manager Web Interfaces and others, taking if needed further actions, including to warn system administrators with local server syslog calls or e-mails.

The processes status of Apache Tomcat are checked using the builtin Apache Tomcat HTTP monitoring interface called Manager.

Usually this interface output HTML but with the undocumented but active in the code of StatusManagerServlet.java functionality to output well formed XML with the manager status data, tomcat-check parses this XML to do its job.

tomcat-check can be complemented with the google code project psi-probe an enhanced replacement for Apache Tomcat Manager.
